
1. Этот старый студенческий репозиторий был выбран в качестве примера и
 клонирован с удалённого на гитхабе на локальный репозиторий. В данном 
репозитории содержатся скрипты на sql для мебельного магазина.
2. Далее создан этот файл, в котором и описаны все действия для данной 
практической работы.
3. После внесения изменений я сохранил его, затем командой git add добавил в
ветку локального репозитория, закоммитил командой git commit и командой
git push отправил на удалённый в гитхаб.
4. Далее выбрал файл 'Create Materials.sql' в папке Script. Добавил 2
переменные: MaterialCountry(страна-поставщик материала) и MaterialStatusImport
(статус материала импортный или нет).
5. Далее создал новую ветку командой git checkout -b upgreat_sql, далее
добавил туда обновленный скрипт-запрос, закомитил и запушил. Для пуша 
использовал команду git push -u origin upgreat_sql, так как в гитхабе ещё не
было этой ветки.
6. Автоматом в гитхабе у меня высветилась кнопка-подсказка для реквеста.
7. Я сделал реквест, а затем смержевал, чтобы объединить с веткой master.
8. Ну и наконец в локальном репозитории переключился на ветку master и сделал 
команду git pull для обновления данных из удалённого репозитория в локальный.
